# Assumptions & Constraints — Website Redesign

## Assumptions

| ID | Assumption |
|----|------------|
| A01 | Brand guidelines will be finalised and approved by 2026-05-20 at the latest. |
| A02 | Contentful enterprise licence will be procured within the first 2 weeks of the project. |
| A03 | Marketing team will be available for UAT during the week of 2026-08-24. |
| A04 | The legacy CMS can export content in a machine-readable format (JSON/CSV) for migration scripting. |
| A05 | HubSpot account and API access will remain unchanged throughout the project. |
| A06 | The agreed page inventory (v1.2) is final — no new pages will be added after M2 (IA sign-off). |
| A07 | Sarah (project owner) has authority to approve design sign-offs and escalate blockers. |
| A08 | Hosting infrastructure (provider TBD) can support the chosen frontend framework with adequate CDN coverage. |

## Constraints

| ID | Constraint |
|----|------------|
| C01 | Hard deadline: 2026-09-30 — aligns with a planned marketing campaign launch. |
| C02 | Team size is fixed — no additional headcount approved mid-project. |
| C03 | Budget is fixed (amount TBD — see budget.md). No scope expansion without formal change request. |
| C04 | IE11 is not supported; minimum browser versions defined in requirements.md. |
| C05 | The customer portal (logged-in experience) is explicitly out of scope for this project. |
| C06 | All work must comply with WCAG 2.1 AA — this is a legal requirement in key markets. |
