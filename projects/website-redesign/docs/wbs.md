# Work Breakdown Structure — Website Redesign

## 1. Discovery & Research (May 2026)
  - 1.1 Stakeholder interviews
  - 1.2 Analytics review (current site performance baseline)
  - 1.3 SEO audit (existing rankings, backlinks, keyword gaps)
  - 1.4 Competitor & inspiration review
  - 1.5 Content inventory (page count, ownership, migration complexity)
  - 1.6 Information architecture draft + stakeholder sign-off

## 2. Design (June – mid July 2026)
  - 2.1 Design system v1 (colours, typography, spacing, components)
  - 2.2 Wireframes — all page templates
  - 2.3 High-fidelity mockups — all page templates
  - 2.4 Interactive prototype (key user journeys)
  - 2.5 Brand sign-off and design freeze

## 3. CMS Setup (June – July 2026, parallel with design)
  - 3.1 Contentful space setup and content modelling
  - 3.2 Editorial workflow configuration
  - 3.3 Content migration scripts (legacy CMS → Contentful)
  - 3.4 Training for marketing team

## 4. Development (mid July – August 2026)
  - 4.1 Frontend scaffold (framework, CI/CD pipeline, environments)
  - 4.2 Design system implementation (component library)
  - 4.3 Page template development (all templates)
  - 4.4 CMS integration (Contentful SDK wired to templates)
  - 4.5 HubSpot form integration
  - 4.6 GA4 + Hotjar instrumentation
  - 4.7 SEO implementation (meta, sitemap, structured data)
  - 4.8 Redirect map implementation
  - 4.9 Accessibility fixes and automated a11y checks in CI

## 5. Testing & QA (September 2026)
  - 5.1 Cross-browser testing
  - 5.2 Responsive / mobile testing
  - 5.3 Performance testing (Lighthouse, Web Vitals)
  - 5.4 Accessibility audit (automated + manual)
  - 5.5 SEO validation (redirect map, sitemap, structured data)
  - 5.6 HubSpot integration testing
  - 5.7 Stakeholder UAT on staging

## 6. Content Migration (August – September 2026)
  - 6.1 Content migration into Contentful
  - 6.2 Content review and copy-editing
  - 6.3 Image optimisation and asset upload

## 7. Launch (late September 2026)
  - 7.1 DNS cutover
  - 7.2 Redirect verification
  - 7.3 Monitoring setup (uptime, error alerting, GA4 live check)
  - 7.4 Post-launch stabilisation (5-day period)
  - 7.5 Project close-out and lessons learned
